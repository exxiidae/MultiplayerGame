Это мой первый полноценный игровой проект. Это многопользовательская 3D-игра в жанре космического шутера, разработанная с нуля с использованием современных веб-технологий. ## 🎮 О проекте

Этот проект родился из желания создать не просто очередную игру, а полноценный сервис, объединяющий игровой процесс с современной серверной инфраструктурой. Идея заключалась в том, чтобы пройти весь путь — от разработки игровой логики и 3D-рендеринга до настройки базы данных и деплоя приложения в облако.

## ✨ Основные возможности

*   **Мультиплеерная игра в реальном времени:** Игроки подключаются к общим игровым сессиям (лобби) и видят действия друг друга в реальном времени благодаря WebSockets.
*   **Три уникальных класса кораблей:**
    *   **Scout:** Легкий и быстрый, но быстро перегревается.
    *   **Assault:** Тяжелый, бронированный, с высоким уроном.
    *   **LazerShip:** Новый корабль с мощным лазером, для которого требуется 1 секунда зарядки, способный уничтожить цель с одного выстрела.
*   **Динамический игровой мир:**
    *   Астероиды спавнятся волнами, двигаются в сторону игрока и наносят урон при столкновении.
    *   Уничтожение астероидов приносит игроку очки.
*   **Система боя:** Игроки могут стрелять друг в друга. При попадании противник получает урон, а в случае уничтожения, он возрождается через 5 секунд с эффектом взрыва.
*   **Система прогресса и сохранения:**
    *   Статистика каждого игрока (количество игр, побед, убийств, уничтоженных астероидов, очки) сохраняется в облачной базе данных **PostgreSQL** на **Render**.
    *   При входе в игру игрок видит свою накопленную статистику, которая обновляется в реальном времени.
*   **Полноценный игровой интерфейс (HUD):**
    *   Отображение здоровья (HP) и шкалы перегрева оружия.
    *   Счетчик очков и количества уничтоженных астероидов.
    *   Информативное меню лобби для выбора корабля и создания/подключения к игровым сессиям.
*   **Оптимизация производительности:** Реализован пул объектов (BulletPool) для пуль и переиспользование геометрии астероидов, чтобы избежать фризов и утечек памяти.

## 🛠 Технологический стек

*   **Frontend:**
    *   **Three.js** — для 3D-рендеринга, работы с моделями, текстурами и анимацией.
    *   **Socket.IO Client** — для двусторонней связи с сервером в реальном времени.
    *   **Bootstrap** — для стилизации интерфейса лобби и элементов HUD.
*   **Backend:**
    *   **Node.js** и **Fastify** — для создания высокопроизводительного HTTP-сервера и обработки API-запросов.
    *   **Socket.IO Server** — для управления подключениями, игровыми комнатами и синхронизацией состояния игры.
    *   **PostgreSQL** — как основная реляционная база данных для сохранения данных игроков.
    *   **pg (node-postgres)** — для подключения и выполнения SQL-запросов к базе данных.
    *   **dotenv** — для безопасного управления конфиденциальными данными (подключение к БД).
*   **DevOps и Хостинг:**
    *   **Git** и **GitHub** — для контроля версий и хранения кода.
    *   **Render** — для деплоя серверного приложения и самой базы данных PostgreSQL в облаке.

## 🧠 Архитектура и ключевые решения

В процессе работы над проектом я столкнулся с рядом вызовов и старался найти элегантные решения:

*   **Синхронизация состояния:** Реализована через обмен сообщениями `move` и `shoot`. Каждый игрок отправляет свои координаты и поворот, а сервер транслирует их остальным участникам сессии.
*   **Модульность:** Весь код клиента и сервера разбит на логические модули: `core/`, `entities/`, `config/` и т.д., что упрощает поддержку и добавление новых функций.
*   **Работа с моделями:** Для загрузки игровых моделей (GLTF) используется `ModelLoader`, который, в том числе, позволяет масштабировать и поворачивать модели прямо в коде без необходимости их редактирования в сторонних программах.
*   **База данных:** Схема базы данных спроектирована для хранения как основных данных об игроках, так и истории их сессий, что открывает возможности для создания таблиц лидеров и расширенной аналитики.
